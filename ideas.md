# 《校园智行者》Web 版设计理念

## 选定设计方案：现代教育科技风格

### 设计哲学
采用**现代教育科技风格**，融合学术严谨与青春活力。设计强调**清晰的信息层级、流畅的交互反馈、以及视觉化算法演示**，使复杂的计算思维概念变得直观易懂。

### 核心设计原则
1. **可视化优先**：算法执行过程通过动画实时展示，让学生直观感受代码逻辑。
2. **游戏化体验**：采用渐进式关卡解锁、实时得分反馈，营造竞争与成就感。
3. **学术严谨**：保留算法复杂度、数据结构等学科术语，强化计算思维教育价值。
4. **响应式设计**：完美适配桌面、平板、手机等多种设备。

### 色彩哲学
- **主色调**：深蓝色（#1e40af）+ 青绿色（#0891b2）
  - 深蓝象征学术与信任，青绿代表活力与创新。
- **辅助色**：
  - 成功绿（#16a34a）：表示正确答案、算法完成。
  - 警告橙（#ea580c）：表示错误、需要注意的步骤。
  - 中性灰（#6b7280）：用于次要信息、禁用状态。
- **背景**：浅色系（#f9fafb），保证文字可读性，减少眼睛疲劳。

### 布局范式
- **首页**：非对称布局，左侧为大型英雄区域（展示游戏主视觉），右侧为关卡卡片网格。
- **游戏页**：上方为算法演示区（占 60% 高度），下方为交互控制区与题目区。
- **导航**：顶部导航栏 + 侧边栏（移动端收起为汉堡菜单）。

### 签名元素
1. **算法可视化卡片**：每个关卡都有一个动态的算法演示区，使用网格、箭头、颜色变化展示数据变化。
2. **进度环形图**：显示玩家的关卡进度与总体得分，采用圆形进度条设计。
3. **代码片段展示**：在关卡详情中嵌入简化的 Python 代码，帮助学生理解算法逻辑。

### 交互哲学
- **即时反馈**：每次点击、输入都有视觉反馈（按钮按下效果、输入框聚焦状态）。
- **流畅动画**：算法演示采用 0.3-0.5s 的缓动动画，既不过快（难以跟踪），也不过慢（显得卡顿）。
- **渐进式披露**：复杂信息分步展示，避免一次性信息过载。

### 动画指南
- **算法演示**：数据元素移动采用 `ease-in-out` 缓动，颜色变化采用 `ease-out` 快速响应。
- **页面切换**：采用 `fade` + `slide-up` 组合，营造轻盈感。
- **按钮交互**：`hover` 时缩放 1.05 倍，`active` 时缩放 0.95 倍，增强触觉反馈。
- **成功动画**：完成关卡时，采用 `scale + fade` 的庆祝动画，配合音效提示。

### 排版系统
- **标题字体**：使用 Google Fonts 的 **Poppins**（粗体 700）用于大标题，传达现代感与活力。
- **正文字体**：**Inter**（常规 400）用于正文，**中等 500** 用于强调。
- **代码字体**：**Fira Code** 用于代码片段展示，保证等宽对齐。
- **字体层级**：
  - H1: Poppins 700, 32px（首页标题）
  - H2: Poppins 600, 24px（关卡标题）
  - 正文: Inter 400, 16px
  - 小文本: Inter 400, 14px
